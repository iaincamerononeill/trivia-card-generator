Accessibility Statement

Last updated: February 11, 2026

Our Commitment

We are committed to making this Trivia Card Generator accessible to all users, including those with disabilities. We strive to meet Web Content Accessibility Guidelines (WCAG) 2.1 Level AA standards.

Accessibility Features

  • Keyboard Navigation - All interactive elements can be accessed via Tab, Enter, and arrow keys
  • Screen Reader Support - Semantic HTML with proper heading hierarchy and ARIA labels
  • High Contrast - Clear visual distinction between elements with sufficient color contrast
  • Responsive Design - Works on mobile, tablet, and desktop devices with flexible layouts
  • Clear Labels - All form inputs have associated labels and helpful descriptions
  • Error Messages - Clear, descriptive error messages for validation and processing errors
  • Focus Indicators - Visible focus states on all interactive elements
  • No Flashing Content - No content that could trigger seizures or vestibular disorders
  • Skip Links - Ability to skip repeated content (where applicable)
  • Resizable Text - Text can be resized up to 200% without loss of functionality

Browser & Assistive Technology Support

This application has been tested with:

  • Screen Readers: NVDA (Windows), JAWS (Windows), VoiceOver (macOS/iOS), TalkBack (Android)
  • Browsers: Chrome, Firefox, Safari, Edge (latest versions recommended)
  • Operating Systems: Windows 10/11, macOS, iOS, Android
  • Zoom: Up to 200% zoom without horizontal scrolling

How to Use with Assistive Technology

Keyboard Shortcuts

  • Tab - Navigate between interactive elements
  • Shift + Tab - Navigate backwards
  • Enter / Space - Activate buttons and controls
  • Arrow Keys - Navigate within radio button groups

Screen Reader Tips

  • Form fields are clearly labeled with their purpose and requirements
  • Required fields are marked with an asterisk (*) and announced as required
  • Error messages are announced when validation fails
  • Status updates are announced during file processing
  • Radio button groups are properly grouped and labeled
  • The page has a logical heading structure (H1 ‚Üí H2 ‚Üí H3)

Print Accessibility

The generated cards support multiple printing options to accommodate different needs:

  • Single-Sided - Easier to read and handle for some users
  • Double-Sided - More compact, environmentally friendly
  • Large Print - Cards are designed with clear, readable fonts (9pt body, 10pt headers)
  • High Contrast Colors - Distinct colors for different categories (Yellow, Green, Pink, Brown, Blue, Orange)
  • Color Independence - Category information is also conveyed through text labels, not color alone

Educational Accessibility

Features designed for inclusive learning:

  • Flexible Subjects - Customize to match curriculum or interests
  • Multiple Difficulty Levels - Adapt to different learning stages (Primary, Secondary, Adult)
  • Clear Visual Layout - Color-coded categories with consistent positioning
  • Tactile Elements - Physical cards support kinesthetic learning
  • AI Generation - Create customized content for specific learning needs

Standards Compliance

We aim to conform to:

  • WCAG 2.1 Level AA - Web Content Accessibility Guidelines (target compliance)
  • Section 508 - US federal accessibility standards
  • EN 301 549 - European accessibility standard
  • UK Equality Act 2010 - Digital accessibility requirements

WCAG 2.1 Conformance Status

  • Level A: Fully conformant
  • Level AA: Partially conformant (PDF output is limitation)
  • Level AAA: Not evaluated

Technical Specifications

  • Semantic HTML5 - Proper document structure with meaningful elements
  • ARIA Labels - Used where native HTML is insufficient
  • CSS for Visual Presentation - Separation of content and style
  • Progressive Enhancement - Core functionality works without JavaScript for file uploads
  • Responsive Grid - Flexible layouts that adapt to screen size
  • Color Contrast - Minimum 4.5:1 for normal text, 3:1 for large text

Known Third-Party Limitations

  • AI Provider Interfaces - Accessibility of OpenAI, Anthropic, and Google sign-up pages is controlled by those providers
  • Browser Behavior - Some accessibility features depend on browser support
  • PDF Viewers - PDF accessibility depends on the viewer used (Adobe Reader, browser built-in, etc.)

Enforcement & Complaints

If you believe we have not met accessibility standards despite our best efforts, please let us know. We take accessibility seriously and will work to resolve issues promptly.
